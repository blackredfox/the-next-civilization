# The Next Civilization

The Next Civilization is an independent publication website about AI, robotics,
institutions, capital, education, and human adaptation. The project is now a
standard React/Vite application and does not depend on Lovable for local
development, builds, or production deployment.

## Tech Stack

- React 19
- Vite 8
- TanStack Start and TanStack Router
- TanStack Query
- Tailwind CSS 4
- Radix UI and shadcn-style UI components
- Nitro for the production Node.js server bundle
- TypeScript, ESLint, and Prettier

## Development

Install dependencies with npm:

```sh
npm ci
```

Start the local development server:

```sh
npm run dev
```

The app runs on Vite's dev server at `http://localhost:3000`.

## Build

Run lint and TypeScript checks:

```sh
npm run check
```

Create a production build:

```sh
npm run build
```

Start the production server bundle locally:

```sh
npm run start
```

The production server entry is generated by Nitro at `.output/server/index.mjs`.

## Deploy

Render should be configured as a Node.js Web Service:

- Build command: `npm ci && npm run build`
- Start command: `npm run start`
- Runtime: Node.js 22.12 or newer
- Environment: set application secrets in Render, not in the repository

Render provides the `PORT` environment variable for web services. The Nitro
Node server reads the platform port at runtime.

Newsletter signup uses Loops from the server-side `/api/subscribe` route.
Configure these Render environment variables:

- `LOOPS_API_KEY`: required Loops API key.
- `LOOPS_MAILING_LIST_ID`: optional Loops mailing list ID for newsletter signups.

For `thenextcivilization.ai`, add the domain in the Render service settings,
configure DNS with the domain provider, then verify the domain in Render. Keep
the GitHub branch deployable: every pushed commit should pass `npm run check`
and `npm run build`.

## Folder Structure

```text
.
├── src/
│   ├── assets/          Static images used by the site
│   ├── components/      Shared React components and UI primitives
│   ├── hooks/           Reusable React hooks
│   ├── lib/             Small framework/runtime helpers
│   ├── routes/          TanStack Router file routes
│   ├── routeTree.gen.ts Generated route tree
│   ├── router.tsx       TanStack Router setup
│   ├── server.ts        TanStack Start server entry wrapper
│   ├── start.ts         TanStack Start middleware setup
│   └── styles.css       Tailwind CSS theme and global styles
├── components.json      shadcn UI component configuration
├── eslint.config.js     ESLint flat config
├── package.json         npm scripts and dependencies
├── tsconfig.json        TypeScript compiler configuration
└── vite.config.ts       Vite, TanStack Start, Tailwind, and Nitro config
```

## Notes

- Lovable can still be used externally for UI prototyping, but this repository
  does not require Lovable metadata, packages, or Vite wrappers.
- Do not edit `src/routeTree.gen.ts` manually; TanStack Start regenerates it.
- Keep generated output directories such as `.output`, `.nitro`, `.tanstack`,
  and `dist` out of version control.
